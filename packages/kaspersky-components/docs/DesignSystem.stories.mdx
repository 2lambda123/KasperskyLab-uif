# Дизайн Система

Описание подходов в использовании дизайн системы. Ресурсы находятся по пути `design-system/`

[Ссылка на дизайн систему в Figma](https://www.figma.com/file/9hiN2DbkqbxbhR9EWRI1VK/B2B-Kit)

## Токены

1. Выгрузка цветовой палитры из фигмы производится в файл `tokens/figma-colors-tokens.json`.
2. На его основании gulp task - updateColors строит файл с токенами цвета `tokens/palette.ts`.
3. Для каждой темы, на основании токенов, необходимо создать файл c псевдонимами (aliases). Например, `themes/light/colors.ts`.
4. Если компонент изпользует цветовую палитру для него необходимо создать отдельный файл в папке `themes/` для каждой темы. Например, `themes/light/button.ts`
5. Для стилизации компонента создаются объекты с цветами и размерами (если необходимо). Если у компонента несколько состояний, например, active, focus, pressed - объект имеет вид:
```
const component = {
  active: {
    ...
  },
  focus: {
    ...
  },
  pressed: {
    ...
  }
}
```

### Структура

- `tokens/palette.ts` - Глобальные токены
- `themes/*theme*/colors.ts` - Aliases
- `themes/*theme*/*component*` - Токены компонент
- `themes/light` - Базовая тема. Все другие темы расширяют базовую

### Использование

Для стилизации компонента используются **только Токены компонента**.

1. Если компонент использует дизайн-токены, необходимо создать файл Токенов компонента для каждой темы.
2. Использовать Токены компонента в `Themed Hook` для построения `Css config`.
3. `Component View` прокидывает `Css config` в стилизованные `styled-components` компоненты.
4. Использовать `Css config` для построения `Component css`.
